## Common Issues
### 1. Pieces Desktop App and CLI on macOS: LTM/Workstream Activity Not Updating
Users on macOS are experiencing issues with the Pieces desktop application and CLI where LTM and Workstream Activity are not updating or recording new activities.  Issues range from complete stoppage after a short period to failures generating summaries with 404 errors.  One user reported an inability to open the Pieces Copilot service with an 'Application not found' error. Troubleshooting steps may include checking application installation, restarting the app, and verifying Pieces OS version compatibility.

### 2. PiecesOS integration failures across platforms
Users on Linux, macOS, and Windows report PiecesOS integration issues with desktop apps and CLI. Problems include broken workstream summaries, failure to detect Ollama on Linux, CLI model validation errors on macOS, and missing PiecesOS directories on Windows. Suggested solutions include verifying Ollama installation/path, checking PiecesOS version and logs, and reinstalling PiecesOS/related apps.

**Related Issues:**

### 3. Users unable to sign in across multiple Pieces platforms
Users are reporting issues signing in to Pieces OS on Desktop Application, VS Code, and CLI.  Error messages vary, but all prevent successful login.  This affects Windows and macOS users on Pieces OS versions 4.3.1 and 12.1.0. Further investigation needed to determine root cause and solution.

Users report that the Auto-Generated Context setting in the Pieces desktop app repeatedly reverts to "Medium" even after being manually set to "High". This occurs across different macOS versions and Pieces builds.  The issue persists after app and OS updates. Users are unsure if the "High" setting ever takes effect.
